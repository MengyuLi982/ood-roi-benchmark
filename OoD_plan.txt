Plan B: Post-hoc Safety-aware OoD Evaluation on Lost & Found

1. Experimental Background and Objective

The current GitHub repository already provides an evaluation framework for Lost & Found OoD detection, including multiple OoD scoring methods and several evaluation metrics. The existing pipeline evaluates whether OoD pixels are correctly detected, but it does not distinguish whether an OoD object is safety-relevant in the current scene.

In this experiment, we introduce a post-hoc safety-aware weighting strategy on top of existing OoD score maps. The goal is to convert the raw OoD probability map into a safety-aware OoD risk map without retraining the OoD detection model.

The core idea is:

[
P_{\text{ood}}(x,y) \neq R_{\text{safety}}(x,y)
]

where (P_{\text{ood}}(x,y)) represents the probability that pixel ((x,y)) belongs to an unknown object, while (R_{\text{safety}}(x,y)) represents the safety-aware risk score.

We define:

[
R_{\text{safety}}(x,y)

P_{\text{ood}}(x,y)
\cdot
W_{\text{safety}}(x,y)
]

where:

* (P_{\text{ood}}(x,y)) is the OoD score map produced by an existing OoD method;
* (W_{\text{safety}}(x,y)) is a post-hoc safety relevance map;
* (R_{\text{safety}}(x,y)) is the final safety-aware OoD risk map.

The safety relevance map is defined based on road geometry and image-level distance cues:

[
W_{\text{safety}}(x,y)

M_{\text{road}}(x,y)
\cdot
\left[
0.7L(x,y)+0.3D(x,y)
\right]
]

where:

* (M_{\text{road}}(x,y)) is the road mask;
* (L(x,y)) is the ego-lane relevance;
* (D(x,y)) is the distance relevance.

The ego-lane relevance is computed as:

[
L(x,y)

\exp
\left(
-\frac{(x-x_c(y))^2}{2(\alpha w(y))^2}
\right)
]

where:

* (x_c(y)) is the center of the road region at image row (y);
* (w(y)) is the road width at image row (y);
* (\alpha) controls the width of the ego-lane relevance region.

The distance relevance is approximated using image height:

[
D(x,y)

\left(
\frac{y}{H}
\right)^2
]

where:

* (H) is the image height;
* larger (y) values correspond to pixels closer to the bottom of the image, which are treated as closer to the ego vehicle.

For each OoD component (o_i), we compute the component-level OoD confidence:

[
p_i^{\text{ood}}

\frac{1}{|o_i|}
\sum_{(x,y)\in o_i}
P_{\text{ood}}(x,y)
]

the component-level safety relevance:

[
w_i^{\text{safety}}

\frac{1}{|o_i|}
\sum_{(x,y)\in o_i}
W_{\text{safety}}(x,y)
]

and the final component-level safety-aware risk score:

[
R_i

p_i^{\text{ood}}
\cdot
w_i^{\text{safety}}
]

2. Experimental Steps

Step 1: Use Existing OoD Score Maps

Use the existing cached OoD score maps generated by the repository.

For each image, load the stored score maps from:

results/roi_variants/score_maps/<stem>.npz

The score maps may include methods such as:

* Energy
* MSP
* Entropy
* DINOv2-kNN
* PixOOD
* RbA

Each method provides a raw OoD score map:

[
P_{\text{ood}}^{m}(x,y)
]

where (m) denotes the OoD method.

⸻

Step 2: Normalize OoD Scores

Because different OoD methods may have different score ranges, normalize each score map to ([0,1]):

[
\tilde{P}_{\text{ood}}^{m}(x,y)

\frac{
P_{\text{ood}}^{m}(x,y)-P_{\min}^{m}
}{
P_{\max}^{m}-P_{\min}^{m}+\epsilon
}
]

where (P_{\min}^{m}) and (P_{\max}^{m}) can be computed using percentile-based normalization.

⸻

Step 3: Generate the Safety Relevance Map

For each image, compute:

[
W_{\text{safety}}(x,y)

M_{\text{road}}(x,y)
\cdot
\left[
0.7L(x,y)+0.3D(x,y)
\right]
]

This produces a continuous safety relevance map in ([0,1]).

⸻

Step 4: Generate the Safety-aware OoD Risk Map

For each OoD method (m), compute:

[
R_{\text{safety}}^{m}(x,y)

\tilde{P}{\text{ood}}^{m}(x,y)
\cdot
W{\text{safety}}(x,y)
]

This produces a safety-aware OoD risk map.

⸻

Step 5: Compute Safety-weighted Metrics

In addition to the original OoD metrics, compute safety-aware metrics.

Safety-weighted OoD Error

[
E_{\text{SW}}

\sum_{x,y}
W_{\text{safety}}(x,y)
\cdot
\left|
\tilde{P}_{\text{ood}}(x,y)

Y_{\text{ood}}(x,y)
\right|
]

where (Y_{\text{ood}}(x,y)) is the ground-truth OoD mask.

Safety-weighted Missed Risk

After thresholding the OoD score map into a binary prediction (\hat{Y}_{\text{ood}}), compute:

[
\text{SW-Miss}

\frac{
\sum_{x,y}
W_{\text{safety}}(x,y)
\cdot
Y_{\text{ood}}(x,y)
\cdot
\left[
1-\hat{Y}{\text{ood}}(x,y)
\right]
}{
\sum{x,y}
W_{\text{safety}}(x,y)
\cdot
Y_{\text{ood}}(x,y)
+
\epsilon
}
]

This metric measures how much safety-relevant OoD area is missed by the detector.

⸻

Step 6: Compute Component-level Safety Ranking

For each connected OoD component (o_i), compute:

[
p_i^{\text{ood}}

\frac{1}{|o_i|}
\sum_{(x,y)\in o_i}
\tilde{P}_{\text{ood}}(x,y)
]

[
w_i^{\text{safety}}

\frac{1}{|o_i|}
\sum_{(x,y)\in o_i}
W_{\text{safety}}(x,y)
]

[
R_i

p_i^{\text{ood}}
\cdot
w_i^{\text{safety}}
]

Then rank OoD components according to:

1. raw OoD confidence (p_i^{\text{ood}});
2. safety-aware risk score (R_i).

⸻

3. Expected Experimental Effects

The expected effect is that the safety-aware risk map will not simply highlight all unknown regions equally. Instead, it should assign higher scores to OoD regions that are more safety-relevant in the image.

For example:

* an OoD object close to the ego-lane should receive a higher safety-aware risk score;
* an OoD object near the bottom of the image should receive a higher safety-aware risk score;
* an OoD object on the road center should be prioritized over an OoD object near the road boundary;
* an OoD object with high raw OoD confidence but low safety relevance may receive a lower final risk score.

The ranking of OoD detections may change after applying the safety relevance map.

Raw OoD score:

[
P_{\text{ood}}(x,y)
]

Safety-aware OoD score:

[
R_{\text{safety}}(x,y)

P_{\text{ood}}(x,y)
\cdot
W_{\text{safety}}(x,y)
]

This allows the experiment to show the difference between unknownness and safety-aware risk.
